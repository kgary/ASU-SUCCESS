Overview

Since the early days of computing, humans wondered if computers could be made "intelligent". The father of modern day computing Alan Turing (movie: The Imitation Game) proposed the Turing test, or the argument that if a computer could communicate like a human, then it possessed human intelligence. To demonstrate this philosophical approach, a computer program named ELIZA was created to emulate a Rogerian psychological therapist to a person by responding to queries. In essence, Eliza was ChatGPT before ChatGPT was!

Activities

For these activities we are giving you your own Eliza! The file eliza_local.html has a self-contained Eliza program, no web server needed.

Pro tip: A software engineering principle is incremental problem solving; each of the numbered items under each Activity are independent, so “code a little test a little” by getting each one done and saving it. Then tackle the next step!

Another pro tip: You are starting from a program I give you and modifying it to make it better. It is tempting to want to go straight to the code and understand it, but unless you are very experienced, you won't know where to start and it will be jibberish. Instead, run Eliza a few times and make an outline of the algorithm to so you understand the process first! This is another software engineering technique called reverse engineering.

  • Activity 1: Modify the HTML
    Let’s personalize your Eliza! Try some of the following:
    1. Change the greeting when the page first runs.
    2. Add a new paragraph using a p tag to have Eliza give a longer introduction to herself. Consider using various formatting mechanisms like you would in Google Docs.
    3. What does Eliza look like? Find a picture on the Internet and insert it in your Eliza welcome message using the image tag.
  • Activity 2: Add some Javascript
    Let’s make your Eliza more intelligent by modifying what she knows. What she knows is stored in the dict variable at the bottom of the document.Try some of the following:
    1. For each existing key/phrases in the dictionary, add more keywords and more phrases
    2. Expand Eliza's vocabularly by adding new - keyword and phrase arrays (pairings)
    3. Can you think of ways to modify what Eliza says to make her sounds more human
    4. (Challenge!) Have Eliza say something to restart the conversation if the user does not type in a response in 20 seconds. To do this challenge, you need to use a new type of event called a timer event.
  • (Advanced) Add new features to Eliza!
    See if you can do the following:
    1. If the user inputs just the word “quit” when talking to Eliza, reset the program to the beginning (without re-loading the webpage).
    2. Change to an ASU SUCCESS theme! Eliza has some pretty boring categories. Make an ASU-inspired theme by changing Eliza to (these are just examples):
      • Vecna – answers all things about the Upside Down
      • Your Dad – seems to be all-knowing about a significant number of things
      • Your Mom – expresses a lot of empathy for anything you say
      • Your Dog - just what is your puppy thinking?
      • A favorite movie character - Mine is Kevin from Despicable Me!
      Or whatever you want! Put images, phrasings, or anything in-character (be nice!). You can even make it a secret friend and you can see if you can get the friend to guess who yours is!
    3. Make Eliza more intelligent. Can you think of ways to make Eliza more intelligent? Yes Activity 2 does that in some ways, but what else can you think of? Keep in mind the Turing test – the idea is to trick someone into believing your computer program is actually a human! What kind of human traits can you inject that might trick the user into thinking Eliza is ALIVE? Make a short (3 minutes or less) YouTube video of you running a Turing test of your (modified) Eliza versus a human and see if you can trick your user!

This activity, while short, includes some very real things about computing.
